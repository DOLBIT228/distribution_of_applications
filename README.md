# Розподіл заявок Bitrix24 (Streamlit)

Це Streamlit-скрипт для рівномірного розподілу заявок між менеджерами у вказаному статусі конкретної воронки Bitrix24.

## Що робить

- Вхід в локальний інтерфейс (логін/пароль з `st.secrets`).
- Логін у CRM не потрібен: зв'язок з Bitrix24 працює через ваш webhook.
- Вибір напрямку (воронка + статус + наступний статус).
- Показ кількості доступних угод у статусі.
- Вибір менеджерів для розподілу.
- Вибір кількості угод для розподілу.
- Автоматичне оновлення відповідального (`ASSIGNED_BY_ID`) **і** переведення угоди в `next_status_id`.
- Класифікація угод за типом:
  - якщо назва джерела містить `Лендинг` → тип `Сайт`,
  - всі інші джерела → тип `Лендинг`.
- Окремий round-robin розподіл по типах `Сайт` і `Лендинг`.
- Денна таблиця розподілу (по менеджерах і типах), що зберігається у локальній SQLite базі `distribution_history.db`.

## Налаштування

1. Встановіть залежності:

```bash
pip install -r requirements.txt
```

2. Створіть файл `.streamlit/secrets.toml` на базі `.streamlit/secrets.toml.example`.

3. Заповніть:
   - `bitrix.webhook_url` — webhook для API Bitrix24,
   - `[[auth.users]]` — користувачі для входу в інтерфейс (login/password/manager_id),
   - `[[directions]]` — напрямки:
     - `name` — назва,
     - `funnel_id` — ID воронки,
     - `status_id` — поточний статус, з якого беруться заявки,
     - `next_status_id` — наступний статус, куди переводяться заявки після розподілу,
   - `[[managers]]` — менеджери для розподілу (name + id).

## Запуск

```bash
streamlit run app.py
```

## Важливо

- Для `status_id` / `next_status_id` використовуйте технічні значення стадій (наприклад `NEW`, `C2:NEW`).
- Скрипт працює тільки з тими воронками/стадіями, які ви вкажете в `secrets.toml`.
- Денна таблиця зберігається локально у файлі `distribution_history.db`.
